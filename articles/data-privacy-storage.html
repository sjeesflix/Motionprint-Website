Data Privacy & Storage

5 min read
Last updated March 2026

Back to Knowledge Base

Motionprint Ergo is designed with data privacy as a core principle. All motion capture data is processed and stored entirely on your local machine — no recordings, session files, or reports are ever uploaded to external servers. This article explains exactly how your data is handled, where files are saved, and what the one narrow exception to local-only operation is.

Stays on your machine
  • Motion capture recordings (.mvnx)
  • Session files and intermediate data
  • Generated PDF and Excel reports
  • Assessment results and scores
  • Private encryption keys
Sent externally (licence only)
  • Licence key (for validation)
  • Machine fingerprint (for seat tracking)
  • No motion data — ever
  • No reports or file contents
  • Encrypted over HTTPS

1 Motion Data Handling

Every step of the Motionprint Ergo processing pipeline — from MVNX import through joint angle calculation, posture classification, and risk scoring — runs entirely on the user's local machine. No cloud processing occurs at any point.

Local-only processing

When you import an MVNX file, the data is read from your local filesystem and passed directly to the processing pipeline running on your machine. The output — assessment scores, time-series graphs, and report files — is written back to your local output directory. The application does not make any outbound network requests during this process.

What is never transmitted

  • Motion capture recordings — .mvnx files and their contents remain on your device at all times
  • Session files — intermediate processing data, joint angle arrays, and posture classifications are stored locally and never shared
  • Assessment reports — generated PDF and Excel files are saved to your local output directory and are not uploaded anywhere
  • Worker or subject data — any personal or identifying information associated with a recording stays on your machine
No cloud dependency

Motionprint Ergo does not require a cloud connection to process data or generate reports. Once activated, the application works fully offline for up to 30 days without contacting any external server.

2 Local Storage

All output files are written to a local output directory on your machine. The directory structure is organised by assessment type so reports are easy to locate.

Output directory structure

output/
NIOSHreport/
report_.pdf
report_.xlsx
session_data.json
KIMLHCreport/
report_.pdf
report_.xlsx
session_data.json

File formats

  • .mvnx — the original Xsens motion capture recording, read at import and not modified by Motionprint Ergo
  • .json — intermediate structured data generated during processing (joint angles, posture classifications, frame-level scores); stored alongside each report for traceability
  • .pdf — the formatted assessment report, including risk scores, charts, and recommendations
  • .xlsx — the detailed data export, providing full multiplier breakdowns and per-moment data in spreadsheet format
Output directory location

The output directory defaults to a folder within the Motionprint Ergo application data path. You can change the output location in Settings → Storage to any local folder you have write access to. Network drives are supported but not recommended for performance reasons.

3 External Transmission

The only time Motionprint Ergo communicates with an external server is during licence validation. This is the single narrow exception to the local-only principle, and it is limited strictly to the information required to verify your licence.

What is transmitted

  • Licence key — your MP-XXXX-XXXX-XXXX-XXXX key, sent to the Motionprint Licence Server to verify it is valid and active
  • Machine fingerprint — a unique identifier derived from hardware characteristics of your device, used to track seat activation and prevent licence sharing

No file contents, motion data, report data, or personally identifiable information is included in licence validation requests. The request contains only the two fields above.

When validation occurs

  • On first activation of the licence
  • Periodically in the background when the application is running and an internet connection is available (to renew the offline grace period)
  • On explicit re-validation after the offline grace period has elapsed
Firewall note

If your organisation's firewall blocks outbound HTTPS traffic, licence validation will fail. The application requires access to the Motionprint Licence Server on port 443. All other application functionality is fully local and unaffected by firewall restrictions.

4 Security Implementation

Motionprint Ergo applies encryption at two levels: in transit (for licence validation requests) and at rest (for locally stored sensitive data).

Encryption standard
AES-256-GCM is used to encrypt sensitive data both in transit and for locally stored private keys. This is the same standard used in modern banking and enterprise security systems.
Key derivation
Encryption keys for locally stored data are derived from the machine fingerprint. This means that encrypted local data can only be decrypted on the machine it was created on — not if copied to another device.
Private key storage
Private keys are stored in a protected location on the local filesystem with restricted file permissions. They are not stored in user-accessible directories or included in any application logs.
Transit security
All communication with the Licence Server is made over HTTPS (TLS 1.2 or higher). The licence key and machine fingerprint payload is additionally encrypted before transmission.
No telemetry
Motionprint Ergo does not collect usage analytics, crash reports, or diagnostic telemetry. No application behaviour data is transmitted externally.
Summary

Your motion capture data, session files, and reports never leave your machine. The only external communication is a small, encrypted licence validation request containing no motion data. All local sensitive data is protected by AES-256-GCM encryption tied to your device's hardware identity.
