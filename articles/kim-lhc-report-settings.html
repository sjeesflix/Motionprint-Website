KIM-LHC Report Settings

The KIM-LHC Report Settings page is your central configuration hub for all KIM-LHC assessments in Motionprint Ergo. Settings you choose here become persistent defaults applied to every new assessment — whether you need to adjust the load rating table, configure the averaged overview, customize your report layout, or control export filenames. Changes take effect the next time you open or run an assessment.

1 Overview

The settings page is accessed via Sidebar → Settings → Reports tab. At the top of the first card a report-type dropdown lets you select which report methodology to configure — select KIM-LHC here. Switching with unsaved changes triggers a confirmation dialog. All configuration options are displayed as a stacked set of modular cards.

At the bottom of the card stack, an action bar provides a single Save and apply button. Clicking it saves your changes to preferences; the page stays open and the button becomes disabled until you make further changes. There is no dedicated Discard button — if you navigate away with unsaved changes, a confirmation dialog appears asking whether you want to leave; choosing "Yes" discards the changes implicitly.

The settings are organized into four functional modules, each addressing a specific aspect of how KIM-LHC assessments are performed, displayed, and exported.

Settings are global defaults

All settings apply to every KIM-LHC assessment created after you save them. They are stored locally on your system and persist across sessions. If you need different settings for a specific assessment, you can adjust them before running the analysis.

2 Input & Data

The Input & Data module controls which fields appear on the input page and in the final report, allowing you to streamline data collection and hide unnecessary fields.

SettingDefaultDescription
Gender not required Off Toggle to hide the gender field from the input page and all reports. When enabled, calculations use female (conservative) reference values for the load rating throughout the assessment.
Hide operator name Off Toggle to conceal the worker's name from the input page, report, and exports. Helpful for privacy-focused assessments or anonymous workflows.
Hide evaluator name Off Toggle to conceal the assessor's name from the input page, report, and exports.
Gender affects load rating calculations

Gender in KIM-LHC directly affects scoring: men and women have different reference values in the load rating table. When gender is hidden, the app applies female values as the conservative default. See the Understanding the KIM-LHC Assessment article for the default load rating tables.

3 Analysis & Scoring

The Analysis & Scoring module is the most KIM-LHC-specific section. It controls the load rating table, multi-task scoring behavior, and report structure options.

SettingDefaultDescription
Effective load weight scoring Default tables Custom table editor. Edit the load rating scores for men and women across all weight bins. See detailed description below.
Auto-generated critical moments On Toggle automatic detection of peak moments (highest-scoring frame) and sustained risk periods (≥ 1 second of consistently elevated scores) in the Analysis tab. Disable if you prefer to mark critical moments manually.
Generate averaged overview Off When enabled and the assessment has two or more lifting tasks, adds an "Average (all moments)" option to the task selector in Score Overview. Shows averaged scores across all tasks. Disable if tasks differ significantly in weight category or movement pattern.
Findings enabled On Toggle whether the Findings tab appears in the report. When disabled, the tab is hidden entirely.
Findings title "Findings" Text input to rename the Findings tab. The custom title appears in the report tab bar and in exported files. Maximum 50 characters.
Findings sections 5 defaults Drag-and-drop list of section blocks within the Findings tab. Add, rename, reorder, or delete sections. A Reset to defaults button restores the standard five sections.

Load weight scoring table editor

The load weight scoring table lets you override the BAuA default load rating values used in the assessment. The editor shows two rows — Men and Women — across ten weight bin columns. Each cell is an editable integer between 0 and 100, representing the load rating score for that gender and weight category.

The weight bins are: 3–5, 5–10, 10–15, 15–20, 20–25, 25–30, 30–35, 35–40, and >40 kg. A Reset to defaults button restores all values to the BAuA standard reference tables.

The 0–3 kg weight category is non-standard

The 0–3 kg weight category is not part of the official KIM-LHC method. Enable it only if your assessments require scoring very light loads that fall below the standard minimum weight bin.

Generate averaged overview

The averaged overview is an optional aggregate result for multi-task assessments. When this setting is enabled and two or more lifting tasks are defined, the Score Overview tab gains an "Average (all moments)" option in the task selector that shows mean scores across all tasks. This is most useful when tasks involve the same weight category and similar movement patterns. The KIM-LHC method does not define a formal formula for multi-task aggregation, so the averaged overview should be used as a supplementary view rather than a primary result.

Findings tab configuration

The Findings tab (if enabled) is a free-form documentation area. You can customize it fully from this module:

  • Enable/Disable — toggle whether the Findings tab appears in the report.
  • Tab title — text input (max 50 characters) to rename the tab. Default is "Findings".
  • Sections list — add, rename, reorder (drag-and-drop), or delete individual section blocks. Each section has a title and a multi-line text area.
  • Reset to defaults — restores the standard five sections: Current Situation, Observations, Risk Assessment, Recommendations, and Conclusion.

4 Viewport

The Viewport module sets the default 3D viewer layout and camera angle for both the input page and the Analysis tab on the results page, ensuring consistent visualization across your assessments. The same two settings apply independently to each page:

SettingOptionsDescription
Layout Single view, Two horizontal, Two vertical, Four views Controls how many viewports are shown. Single view shows one full-width viewport. Two horizontal shows two viewports side by side. Two vertical stacks them. Four views shows a 2×2 grid.
Camera view Perspective, Front, Back, Left, Right, Top Sets the default camera angle. Perspective is a realistic angled view. The others are orthogonal views useful for checking posture alignment from a specific direction. When a multi-viewport layout is selected, each viewport has its own camera view setting.
Defaults can be overridden

The settings here establish defaults when you first open an assessment or the Analysis tab. Once inside a report, you can always change the layout and camera view interactively, or orbit freely with your mouse. The global settings do not lock the interface.

5 Export

The Export module controls the naming conventions for PDF and Excel files when you export a completed KIM-LHC assessment report, plus one KIM-LHC-specific export behavior option.

Filename templates

PDF and Excel exports each have their own filename template, both defaulting to:

{workstation}_{current_date}_{report_type}

The two templates are independent — you can customize them separately. The file extension (.pdf, .xlsx) is appended automatically. For a full overview of available variables, validation rules, fallback behaviour, and example templates, see Exporting Reports (PDF & Excel).

Export settings

SettingDefaultDescription
Export only average lifting task Off When enabled, exports only the averaged overview instead of individual task results. This setting is disabled (grayed out) unless "Generate averaged overview" is turned on in Analysis & Scoring.
Use company branding Off Toggle to include your company logo and brand colour in exported PDF and Excel reports — on the cover page, section headers, and footer. When disabled, exports display "Motionprint" as the default footer company name.
Company branding is configured in General Settings

The company name, logo, and brand colours are configured in Sidebar → Settings → General, not here. Set them there once and they apply to all branded exports across all report types. See Customization & Branding for a full walkthrough.
