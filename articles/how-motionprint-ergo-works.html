How Motionprint Ergo Works

6 min read
Last updated February 2026

Motionprint Ergo is a Windows desktop application that converts Xsens MVNX motion capture recordings into standardised ergonomic risk assessment reports. No manual video coding, no spreadsheet calculations, no cloud uploads — the entire process runs locally on your machine from raw motion data to finished PDF.

1 The Workflow

The process follows a fixed pipeline: capture motion data with an Xsens suit, export it from Xsens MVN, and import it into Motionprint Ergo for analysis and reporting. Each stage is described below.

  1. 1
    Record the task with an Xsens suit

    The worker wears an Xsens MVN inertial motion capture suit — either MVN Awinda (wireless IMU sensors) or MVN Link (body suit with embedded sensors). The assessor records the lifting or handling task in Xsens MVN Analyze or Animate.

  2. 2
    Export the recording as an .mvnx file

    From Xsens MVN, export the session in MVNX format. This is an XML-based file containing the full body motion data — segment positions, orientations, joint angles, and foot contacts — sampled at up to 60 Hz.

  3. 3
    Open Motionprint Ergo and select a report type

    Launch Motionprint Ergo and create a new assessment. Choose the assessment method — NIOSH Lifting Equation or KIM-LHC — based on the type of task and the standard your organisation follows.

  4. 4
    Import the .mvnx file

    Upload the .mvnx file into the app. Motionprint Ergo validates the file, checks that all required segments and joint data are present, and loads the full motion time-series into memory.

  5. 5
    Define the task parameters

    Trim the recording to the relevant task window, mark one or more lifting moments (individual lift cycles), and enter the remaining task parameters: load weight, lift frequency, daily work duration, and coupling quality. The biomechanical variables (hand distances, posture angles, vertical travel) are derived automatically from the motion data.

  6. 6
    Run the analysis

    Trigger the analysis. The backend processes every frame in the recording and streams real-time progress back to the UI. Depending on recording length, this typically takes a few seconds to under a minute.

  7. 7
    Review results and export

    Results are presented across three tabs: Overview (risk score summary), Analysis (frame-by-frame graphs and posture breakdown), and Findings (actionable recommendations). Export a branded PDF report and/or an Excel spreadsheet for sharing and record-keeping.

2 Under the Hood

Motionprint Ergo runs as a single desktop window, but internally it consists of two components that work together on your local machine.

Flutter desktop UI

The visible application — the window you interact with — is built with Flutter. It handles the user interface, file import, parameter inputs, results display, and report export. This is the only thing you see or need to interact with.

Python analysis backend

Running silently in the background is a Python-based computation engine that listens on localhost:8000. When you trigger an analysis, the Flutter UI sends the motion data and task parameters to this local server, which processes every frame and streams the results back in real time.

Privacy note

No motion data, assessment results, or personal information is ever sent to an external server. All processing happens locally on the user's machine. The only outbound network connection is to the Motionprint licence server for licence validation — no assessment data is transmitted in that process.

3 What the App Produces

A completed assessment in Motionprint Ergo produces several layers of output, from a single headline risk score down to frame-level biomechanical detail.

Risk score

The headline output is a risk score tied to the selected standardised method — a Lifting Index (LI) for NIOSH, or a total risk score for KIM-LHC. Both are mapped to clearly defined risk bands (low, moderate, high, very high) with corresponding action recommendations.

Frame-by-frame time series

Every metric — posture angles, hand distances, derived multipliers, and the rolling risk score — is computed for every frame in the recording. This produces an interactive graph in the Analysis tab, allowing you to scrub through the recording and identify the exact moment a high-risk posture occurred.

Body posture classification

For each frame, the body is classified across multiple regions (trunk, neck, shoulders, arms, legs). These classifications are visualised using anatomical SVG diagrams that show the dominant posture category for each region throughout the assessed lifting moment.

Critical moments

The app automatically identifies frames with the highest risk contribution and marks them as critical moments. You can also add manual critical moment markers at specific points in the recording to call out specific postures or events for the report reader.

PDF and Excel reports

Export a fully formatted PDF report — optionally with a cover page, your company logo, and custom accent color — or an Excel spreadsheet containing the raw calculated values and time-series data for further analysis. See Customization & Branding for how to configure report appearance.

4 Supported Report Types

Motionprint Ergo currently supports two internationally recognised ergonomic assessment methods for manual material handling tasks.

NIOSH Lifting Equation (Revised 1991)

Developed by the US National Institute for Occupational Safety and Health, the NIOSH Lifting Equation calculates a Recommended Weight Limit (RWL) and a Lifting Index (LI) for a defined lifting task. It is standardised under ISO 11228-1 and EN 1005-2 and is widely used across North America, Europe, and internationally.

  • Produces an LI score with low / moderate / high risk classification
  • Supports multiple lifting moments and Composite Lifting Index (CLI) for multi-task assessments
  • Includes optional correction factors: two-person lift, one-hand lift, extended duration (ISO 11228-1)

For a detailed explanation of the method, see Understanding the NIOSH Lifting Assessment.

KIM-LHC (Key Indicator Method – Lifting, Holding, Carrying)

Developed by the German Federal Institute for Occupational Safety and Health (BAuA), the KIM-LHC is a widely used European method for evaluating physical workload in manual material handling tasks. It produces a total risk score based on load weight, working duration, body posture across 10 body regions, and unfavorable working conditions.

  • Produces a total score mapped to four risk levels (green / yellow / orange / red)
  • Scores posture automatically across all 10 body regions from motion data
  • Supports separate or averaged multi-moment assessments

For a detailed explanation of the method, see Understanding the KIM-LHC Assessment.
