How Motionprint Ergo Works

Motionprint Ergo is a comprehensive platform that transforms motion capture data into actionable ergonomic insights. The workflow spans from recording movement with an Xsens suit to generating fully branded reports with validated risk scores — through a pipeline that handles preprocessing, biomechanical analysis, and export entirely on your local machine.

1 Motion Capture Input

The starting point for every assessment is a motion capture recording made with an Xsens MVN system. Motionprint Ergo is built around the MVNX file format — an XML-based container that holds the complete body motion time-series exported directly from Xsens MVN Analyze or Animate.

Recording with Xsens

Motion data is captured using either the MVN Awinda (wireless IMU sensors) or MVN Link (body suit with embedded sensors) system. The assessor records the task — a lifting cycle, carrying task, or handling sequence — in Xsens MVN, then exports the session as an MVNX file. This file contains full-body segment positions, orientations, joint angles, and foot contacts sampled at up to 60 Hz.

Preprocessing

Before analysis begins, Motionprint Ergo runs a two-step preprocessing stage to ensure the data is clean and correctly structured.

  • Trimming — Removes frames outside the relevant task window, focusing the analysis on the actual lifting or handling movement rather than idle time before or after the recording.
  • Validation — Checks data integrity: verifies frame rate consistency, confirms all required body segments are present, and validates coordinate mapping. Clear error messages are shown if the file fails any validation check, so you always know exactly what needs to be corrected.
Supported formats

Motionprint Ergo primarily supports the MVNX format. Preprocessing scripts are also available for trimming recordings and regenerating derived variables when raw exports require adjustment before import.

2 Data Processing

Once the MVNX file is loaded and validated, the analysis backend processes every frame in the recording through a structured pipeline. This runs locally on your machine — no data is sent to any external server.

Peak detection

The pipeline begins by identifying critical moments in the motion data — the frames where biomechanical load is highest. These peaks are the primary reference points for ergonomic risk scoring and are also surfaced as marked moments in the results view, so assessors can immediately navigate to the highest-risk postures in the recording.

Asymmetry analysis

Asymmetry between the left and right sides of the body is computed across every frame. Trunk rotation, lateral bending, and limb asymmetry are quantified and tracked over the duration of the task. This produces the asymmetry time-series visible in the Analysis tab — a frame-by-frame view of how symmetrically the worker performed the task and at which points asymmetric loading peaked.

Ergonomic risk scoring

With peaks and asymmetry scores established, the pipeline applies the selected assessment method — NIOSH Lifting Equation or KIM-LHC — to calculate the final risk scores. Biomechanical variables (hand distances, vertical travel, trunk and limb posture angles) are derived directly from the motion data; only load weight, task frequency, and work duration require manual input.

3 Report Generation

After analysis completes, results are presented across three tabs in the application and can be exported in multiple formats.

Risk scores

The headline output is a validated risk score tied to the selected method — a Lifting Index (LI) for NIOSH assessments, or a total risk score mapped to four levels (green / yellow / orange / red) for KIM-LHC. Both are accompanied by clearly defined risk bands and recommended actions based on the score.

Key metrics

Beyond the headline score, every assessment produces a structured set of metrics that give full visibility into the risk factors driving the result:

  • Critical moments — frames automatically flagged as highest-risk, with optional manual markers you can add at any point in the recording
  • Asymmetry scores — frame-level asymmetry values showing how evenly load was distributed across the body throughout the task
  • Task-specific risk factors — the individual multipliers and sub-scores that make up the final risk score, broken down per body region and task parameter

Customisation

Reports can be branded with your organisation's logo, custom accent colour, and company name. A cover page can be added for client-facing deliverables. See Customization & Branding for the full configuration options.

4 Workflow Automation

For organisations running Motionprint Ergo at scale — or integrating it into larger data pipelines — the platform supports several modes of automated operation beyond the standard GUI workflow.

Batch processing

Multiple MVNX files can be processed simultaneously without manual intervention between each file. Batch mode applies the same assessment configuration across all files in the queue and outputs a report for each, making it practical to process a full day's recordings in a single session.

Silent mode

Silent mode enables fully automated, non-interactive workflows. When invoked, the application processes the input file and generates the output report without displaying any user interface or prompting for input. This is designed for scheduled tasks, scripted pipelines, or server-side automation scenarios.

CLI tools

Advanced users can interact with Motionprint Ergo directly through the command line. The CLI exposes the same analysis pipeline as the GUI — accepting an MVNX file, assessment method, and task parameters as arguments and outputting a report to a specified path. This makes it straightforward to integrate Motionprint Ergo into shell scripts or data processing workflows.

CLI & automation

CLI access and silent mode are available for users on the Professional plan. Contact us if you need to integrate Motionprint Ergo into an existing data pipeline or automation workflow.

5 Output Formats

Every completed assessment can be exported in two structured formats, each suited to a different use case.

PDF reports

The PDF export produces a fully formatted assessment document including the headline risk score, a breakdown of the contributing risk factors, critical moment snapshots, and a findings section with recommended interventions. The report is ready to share with clients, H&S managers, or regulatory bodies without further editing. Branding (logo, colours, company name) is applied automatically based on your account settings.

Excel reports

The Excel export provides structured data with colour-coded highlights for each risk level, making it straightforward to compare results across assessments, build trend charts, or feed the data into your existing H&S management system. The spreadsheet includes the raw calculated values alongside the derived scores for full traceability.

Visualisation

Within the application, results are accompanied by interactive charts and 3D model visualisations for enhanced interpretation. The 3D view lets assessors inspect body posture at any frame in the recording, cross-referenced with the risk score at that moment — making it easy to explain a finding to a worker or a client in concrete, visual terms.
